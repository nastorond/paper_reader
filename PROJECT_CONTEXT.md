# 📚 Portable PDF Reader - Project Context (Handover)

이 문서는 다른 기기(예: Mac)나 다른 환경에서 작업을 이어서 진행할 때, 프로젝트의 전체적인 방향성과 현재까지의 진행 상황을 코파일럿(또는 AI 어시스턴트)이 빠르게 파악할 수 있도록 돕기 위해 작성되었습니다.

## 🎯 프로젝트 개요
* **목표**: 논문을 직관적이고 효율적으로 읽고 관리할 수 있는, 휴대성이 극대화된 크로스플랫폼(Windows/Mac) 데스크톱 PDF 리더.
* **특징**: 단일 실행 파일로 패키징 가능, `papers` 및 데이터 폴더를 JSON(사이드카) 기반으로 관리하여 설정과 논문 데이터의 완벽한 이동성(Portability) 보장.

## 🛠️ 핵심 구현 기능 (현재 상태)
1. **커스텀 UI 기반 연속 스크롤 PDF 뷰어**
   - JavaScript/HTML 기반의 프리미엄 UI 뷰어 적용.
   - 드래그를 통한 텍스트 하이라이트 기능 완성.
2. **참고문헌(Reference) 추출 및 시각화**
   - 논문 내 참고문헌 목록 파싱 및 인용 번호(예: [1], [2]) 매핑 로직 구현.
   - 🌟최근 수정: 넘버링이 없는 논문(엔터 바꿈으로만 구분된 논문)의 레퍼런스들도 `\n\s*\n` 정규식 폴백을 통해 정상적으로 분리되도록 파싱 로직 개선.
   - 최근 수정: 한국어 논문 등에서 목차(TOC)를 참고문헌으로 오인하는 문제 해결을 위한 정규식 고도화.
   - *Semantic Scholar API* 연동: 로컬 PDF 메타데이터를 기반으로 더 정확한 서지 정보 조회, 논문 간 인용 관계(Citation Graph) 추적 지원.
   - 새로 추가된 논문에 대해 UI 상에서 '인덱싱 중'임을 표시하는 Placeholder 피드백 추가.
4. **사용자 탐색 경험 고도화**
   - **히스토리(History)**: 최근 읽은 논문들의 경로 역추적 및 네비게이션 기능.
5. **Git 백업 및 브랜치**
   - 현재 안정화된 코드를 `dev` 브랜치에 안전하게 커밋하여 관리 중.
   - 최근 수정: `papers` 디렉토리 경로 관련 버그 지속 해결(중복 생성 방지).

## 📁 디렉토리/핵심 파일 구조
* `src/main.py`: 애플리케이션의 핵심 백엔드/로직 엔트리포인트 (API 연동, 파일 시스템 통신).
* `src/config.py`: 휴대성 유지를 위한 Configuration 로직 구조(경로 등).
* `src/ui/app.js` & `src/ui/index.html`: 프론트엔드 역할을 담당하는 PDF 렌더링 및 UI 로직.
* `papers_index.json` (데이터 파일): 파싱된 메타데이터 및 설정 등이 보관되어 휴대성 100%를 보장함.
* `LaunchReader.command` (Mac용), `run.bat` (Windows용): 각 OS별 애플리케이션 빠른 실행 스크립트.
